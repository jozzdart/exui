### üëÅÔ∏è `visible` ‚Äî Conditional Visibility for Widgets

Simplify visibility logic in your widget tree with expressive, chainable methods. These extensions replace repetitive ternary conditions and help keep your UI code declarative and clean.

- `visibleIf(bool)` ‚Äî Show this widget only if the condition is `true`; otherwise returns an empty box.
- `visibleIfNot(bool)` ‚Äî Inverse of `visibleIf`.
- `visibleIfNull(Object?)` ‚Äî Show this widget only if the given value is `null`.
- `visibleIfNotNull(Object?)` ‚Äî Show this widget only if the given value is **not** `null`.
- `hide()` ‚Äî Always returns an empty widget (`SizedBox.shrink()`).
- `invisible()` ‚Äî Alias of `hide()` for readability.
- `boxShrink()` ‚Äî Returns `SizedBox.shrink()` directly.

All methods return valid widgets and are safe to chain inside build methods.

#### üß™ Examples

```dart
"Welcome".text().visibleIf(isLoggedIn); // Show only if condition is true
```

```dart
"Not logged in".text().visibleIfNot(isLoggedIn); // Show only if condition is false
```

```dart
"Missing info".text().visibleIfNull(user); // Show only if value is null
```

```dart
"Edit profile".text().visibleIfNotNull(user); // Show only if value is not null
```

```dart
"Hidden message".text().hide(); // Always hidden
```

```dart
"Not visible".text().invisible(); // Same as hide(), for clarity
```

```dart
final emptyBox = "Unused".text().boxShrink(); // Returns an empty widget directly
```

> üí° **Why use this?**
> Instead of writing:
>
> ```dart
> condition ? MyWidget() : const SizedBox.shrink()
> ```
>
> Just write:
>
> ```dart
> MyWidget().visibleIf(condition)
> ```

> üîí These methods **never break layout structure** ‚Äî they return a valid widget in all cases and help you write safer conditional UI.

### üå´Ô∏è `opacity` ‚Äî Control Widget Transparency

Quickly apply opacity to any widget using clean, expressive methods. These extensions eliminate the need to wrap widgets manually in `Opacity` and support percentage-based and common preset values.

- `opacity(double)` ‚Äî Set widget transparency using a `0.0‚Äì1.0` value.
- `opacityPercent(double)` ‚Äî Use percentage (`0‚Äì100`) for readability.
- `opacityHalf()` ‚Äî Set opacity to 50%.
- `opacityQuarter()` ‚Äî Set opacity to 25%.
- `opacityZero()` ‚Äî Set opacity to 0.
- `opacityTransparent()` ‚Äî Alias of `opacityZero()`.
- `opacityInvisible()` ‚Äî Alias of `opacityZero()`.

All methods return a wrapped `Opacity` widget and are safe to chain with other extensions.

#### üß™ Examples

Set to `70%` visible

```dart
"Hello".text().opacity(0.7);
```

Set to `40%` using percent

```dart
"Faded".text().opacityPercent(40);
```

```dart
"50/50".text().opacityHalf(); // Half visible (0.5)
```

```dart
"Subtle".text().opacityQuarter(); // Quarter visible (0.25)
```

> üí° Instead of writing:
>
> ```dart
> Opacity(
>   opacity: value,
>   child: MyWidget(),
> )
> ```
>
> Just write:
>
> ```dart
> MyWidget().opacity(value)
> ```
>
> Use `.opacityPercent()` when working with designer specs or to make your code more intuitive at a glance.

### üî£ `icon` ‚Äî Quickly Create and Style Icons

### üìè `padding` ‚Äî Add Padding to Any Widget

Apply padding effortlessly with readable, chainable methods. These extensions wrap your widget in a `Padding` widget using concise, expressive syntax.

- `padding(EdgeInsets)` ‚Äî Use any `EdgeInsets` object directly.
- `paddingAll(double)` ‚Äî Uniform padding on all sides.
- `paddingOnly({left, top, right, bottom})` ‚Äî Custom padding per side.
- `paddingSymmetric({horizontal, vertical})` ‚Äî Padding for x/y axes.
- `paddingHorizontal(double)` ‚Äî Shorthand for horizontal-only padding.
- `paddingVertical(double)` ‚Äî Shorthand for vertical-only padding.
- Per-side utilities:
  - `paddingLeft(double)`
  - `paddingRight(double)`
  - `paddingTop(double)`
  - `paddingBottom(double)`

All methods return a wrapped `Padding` widget and can be freely chained with other extensions.

#### üß™ Examples

```dart

"Hello".text().paddingAll(12);
```

```dart

"Box".text().paddingSymmetric(horizontal: 16, vertical: 8);
```

```dart
// 10px padding only on the left
"Back".text().paddingLeft(10);
```

```dart
// Custom per-side padding
"Margin".text().paddingOnly(left: 8, top: 4, bottom: 12);
```

> üí° **Why use this?**
> Instead of writing:
>
> ```dart
> Padding(

> )
> ```
>
> Just write:
>
> ```dart

> ```

### ‚ûñ `margin` ‚Äî Add Outer Spacing Around Widgets

Add clean, configurable margins around any widget with chainable extensions. These methods wrap the widget in a `Container` with `margin`, avoiding cluttered layout nesting and improving code clarity.

- `margin(EdgeInsets)` ‚Äî Use any `EdgeInsets` object directly.
- `marginAll(double)` ‚Äî Equal margin on all sides.
- `marginOnly({left, top, right, bottom})` ‚Äî Custom margin per side.
- `marginSymmetric({horizontal, vertical})` ‚Äî Horizontal & vertical margin.
- `marginHorizontal(double)` ‚Äî Shorthand for horizontal-only margin.
- `marginVertical(double)` ‚Äî Shorthand for vertical-only margin.
- One-sided margin helpers:
  - `marginLeft(double)`
  - `marginRight(double)`
  - `marginTop(double)`
  - `marginBottom(double)`

All methods return a wrapped `Container` and can be freely chained with other extensions.

#### üß™ Examples

```dart
// 16px margin on all sides
"Card".text().marginAll(16);
```

```dart
// 24px horizontal, 12px vertical
"Tile".text().marginSymmetric(horizontal: 24, vertical: 12);
```

```dart
// 8px margin only on top
"Header".text().marginTop(8);
```

```dart
// Custom side-by-side margin
"Box".text().marginOnly(left: 6, bottom: 10);
```

> üí° **Why use this?**
> Instead of writing:
>
> ```dart
> Container(
>   margin: EdgeInsets.only(left: 8, top: 4),
>   child: MyWidget(),
> )
> ```
>
> Just write:
>
> ```dart
> MyWidget().marginOnly(left: 8, top: 4)
> ```

> ‚öñÔ∏è **Margin vs Padding**  
> Use **`padding`** to add spacing _inside_ a widget's boundary ‚Äî like space around text in a button.  
> Use **`margin`** to add spacing _outside_ a widget ‚Äî like separating it from other widgets.
>
> üü¢ For most layout needs, **`padding` is the preferred and safer default**. Use `margin` when you need to push the widget away from surrounding elements, but be cautious with nesting to avoid layout issues.

### üß≠ `row*` / `column*` ‚Äî Rapid Alignment Extensions for Flex Layouts

Stop repeating alignment boilerplate in your `Row` and `Column` widgets. These expressive extensions let you instantly apply common combinations of `mainAxisAlignment` and `crossAxisAlignment`, all while preserving full layout control. They make **UI creation dramatically faster and more readable**, especially in complex layouts or dynamic widget lists.

#### üß© Functional Groups

All extensions are available for both `Row` and `Column`, following the same structure:

- ‚úÖ `columnMain*` ‚Äî Sets `mainAxisAlignment`, customize others
- ‚úÖ `columnCross*` ‚Äî Sets `crossAxisAlignment`, customize others
- ‚úÖ `column<Main><Cross>()` ‚Äî Aligns both axes (e.g., `columnCenterStart`)

* ‚úÖ `rowMain*` ‚Äî Sets `mainAxisAlignment`, customize others
* ‚úÖ `rowCross*` ‚Äî Sets `crossAxisAlignment`, customize others
* ‚úÖ `row<Main><Cross>()` ‚Äî Aligns both axes (e.g., `rowCenterStart`)

All methods accept **all standard layout parameters**, including:

- `mainAxisSize`

### üßä `stack` ‚Äî Overlay Widgets with Full Stack Control

Build layered UI structures with intuitive, chainable extensions. These methods let you create `Stack` layouts directly from a `List<Widget>` and configure fit modes easily with named variants.

- `stack()` ‚Äî Wrap a list in a standard `Stack` (`StackFit.loose` by default).
- `stackLoose()` ‚Äî Equivalent to `StackFit.loose` (default).
- `stackExpand()` ‚Äî Children fill the available space (`StackFit.expand`).
- `stackPassthrough()` ‚Äî Children keep their intrinsic size (`StackFit.passthrough`).

‚úÖ Fully supports all native `Stack` parameters:

#### üß™ Examples

```dart

[

].stack();
```

```dart

[
  Image.asset("bg.png"),
  "Play".text().positionedBottom(0),
].stackExpand();
```

```dart
// Stack with passthrough sizing
[
  Icons.circle.icon(), // same as Icon(Icons.circle)
  Icon(Icons.star),
].stackPassthrough();
```

```dart
// Stack with custom alignment and clip behavior
[
  ...someWidgets,
].stack(
  alignment: Alignment.center,
  clipBehavior: Clip.none,
);
```

```dart

```

> üí° Instead of writing:
